# Customer Churn Analysis (SQL + Python)
Processed and analyzed 100K+ e-commerce transaction records to study customer behavior and churn patterns.

## Project Overview
This project analyzes customer churn patterns in an e-commerce platform to identify behavioral factors that lead customers to stop purchasing. The goal is to uncover insights that can help businesses improve customer retention and engagement.

## Problem Statement
Customer churn directly impacts business revenue and long-term growth. This project aims to analyze customer transaction data to understand patterns of inactivity, delivery performance, and purchasing behavior that contribute to churn.

## Tools Used
- Python
- SQL
- Pandas
- Seaborn
- Matplotlib

## Dataset
E-commerce transactional dataset containing:
- Customer ID
- Order ID
- Purchase date
- Delivery time
- Payment information
- Order value
- Customer activity metrics

The dataset included **100K+ transaction records** extracted from multiple relational tables.

## Analysis Performed
- Extracted and merged datasets using **SQL joins across 4 relational tables**
- Cleaned and prepared data using **Python (Pandas)**
- Performed **Exploratory Data Analysis (EDA)** to understand customer behavior
- Analyzed **10+ variables related to purchasing activity and delivery performance**
- Created visualizations using **Seaborn and Matplotlib** to identify churn patterns

## Key Insights
- Customers experiencing **delivery delays showed significantly higher churn probability**
- **Purchase inactivity over long periods strongly correlated with churn behavior**
- Frequent buyers demonstrated much **higher retention rates**
- Delivery performance and engagement levels were major indicators of customer retention
